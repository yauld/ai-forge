# 订单报告助手

## 使用场景

当用户要求查询订单状态、商品、金额，并将结果整理成报告时，使用这个 Skill。

如果任务涉及写入文件、提交报告或覆盖已有内容，必须在执行写入前等待人工确认。

## 可用 MCP 工具

- 在 `stage11-order-report` MCP Server 中，有 `get_order` 工具。
- 查询订单时，需要调用 `get_order(order_id)`。

## 执行步骤

1. 从用户请求中提取订单号，例如 `O-1001`。
2. 调用 `stage11-order-report` MCP Server 中的 `get_order` 工具查询订单。
3. 根据工具返回结果生成简短订单报告草稿。
4. 写入文件前暂停，等待人工确认。
5. 只有人工明确批准后，才允许写入报告文件。
6. 如果人工拒绝，不写文件，只返回拒绝原因。

## 输出要求

报告应包含：

- 订单号
- 状态
- 商品
- 金额
- 数据来源

不要编造工具结果中没有出现的信息。
