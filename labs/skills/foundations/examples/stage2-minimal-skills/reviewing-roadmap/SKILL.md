# 路线图审查

使用这个 Skill，把路线图当作学习或执行计划来审查，而不是只润色标题列表。

## 输入

可以接收以下任意组合：

- 路线图文档
- 分阶段学习计划
- 计划中的文章或实验列表
- 已有专题 README
- 用户提供的目标和约束

## 工作流

1. 识别路线图的目标读者、最终目标和默认前置知识。
2. 检查阶段是否从简单概念走向更难的实现细节。
3. 检查每个阶段是否只有一个清晰研究问题。
4. 检查每个阶段是否有具体产出，例如文章、脚本、流程图、清单或可运行实验。
5. 查找缺失的桥接主题，避免读者需要尚未介绍的知识。
6. 查找范围过大的阶段，并建议拆分。
7. 检查状态标签是否一致，例如“已完成”“进行中”“待研究”。
8. 先总结重要结构问题，再给出次要措辞建议。

## 输出结构

```markdown
## 整体判断

## 主要问题

## 阶段顺序建议

## 缺失产出

## 次要措辞建议

## 下一步
```

## 约束

- 优先检查学习顺序和可执行产出，不要只润色标题。
- 不要把路线图扩写成完整文章。
- 不要编造已完成成果；如果文件或实验缺失，标注为计划中或缺失。
- 所有建议都要围绕用户给出的目标。
