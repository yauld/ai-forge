# 周报生成

使用这个 Skill，把零散的周度工作材料整理成一份清晰的周报。

## 输入

可以接收以下任意组合：

- 会议记录
- 任务进展
- 指标快照
- 阻塞和风险
- 下周计划
- 用户提供的周报格式

如果用户没有提供格式，使用下面的默认输出结构。

## 工作流

1. 识别周报周期、团队或项目范围，以及目标读者。
2. 从输入中抽取事实性进展，只保留材料支持的信息。
3. 按项目、工作流或目标分组，选择最便于阅读的组织方式。
4. 区分已完成、进行中、阻塞风险和下周计划。
5. 对缺失或不确定的信息做显式标注，不要用猜测补齐。
6. 用简洁的业务语言改写周报。
7. 检查每个结论是否能回到输入材料。

## 输出结构

```markdown
# 周报

## 本周摘要

## 已完成

## 进行中

## 阻塞 / 风险

## 下周计划

## 待确认
```

## 约束

- 不要编造指标、负责人、日期或结果。
- 不要隐藏不确定信息，统一放入 `待确认`。
- 保留重要的人名、数字、决策和截止时间。
- 如果输入材料太少，输出短周报，并列出缺失信息。
