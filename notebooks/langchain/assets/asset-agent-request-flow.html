Asset Agent 最小闭环

一次资产运营提问
背后的 Agent 链路

从用户在资产中心提问开始，系统把页面上下文、Agent 路由、工具调用、确定性计算和大模型表达串成一个可控闭环。

?
用户在 Argus Copilot 中提问
列出最值得关注的 3 台资产，并说明原因
1
前端携带页面上下文
当前模块、路由、筛选条件、选中资产和 thread_id 一起提交。
↓
2
AI Chat API 接收统一请求
前端只调用统一接口，API 层不绑定具体专业 Agent。
↓
3
Orchestrator 路由到 asset-agent
根据资产关键词、当前模块和选中对象选择资产中心专业 Agent。
↓
4
AssetAgent 组装模型、Prompt、Tools、Memory
使用 LangChain create_agent，并通过 LangGraph InMemorySaver 保留会话上下文。
↓
5
LLM 选择资产工具
“最值得关注”会落到 rank_assets_by_attention 这种结构化工具调用。
↓
6
代码读取数据并计算排序
AssetReader 读取 normalized_assets，AssetRanker 按固定规则生成分数和原因。
↓
7
LLM 基于工具结果组织答案
模型负责表达，不负责猜数据、不负责直接计算资产风险。
核心原则
上下文由前端提供
路由由 Orchestrator 负责
事实由 Tool 获取
计算由代码完成
表达由模型生成
记忆由 LangGraph 维护
