## 1、余弦相似度是什么

以二维向量为例，一个向量不只有“方向”，还有“模长”。模长就是向量的长度。但做语义相似度时，我们更关心方向像不像，不太关心模长大不大。

假设有几段文本被向量化后，得到了下面三个二维向量：

```
A = [1, 1]
B = [10, 10]
C = [5, 0.3]
```

它们在语义空间里的表示类似：
![alt text](<assets/Pasted image 20260530125351.png>)


A 和 B 的数字大小差很多，模长不一样。但它们指向的是同一个方向，所以从余弦相似度的角度看，它们可以认为很相似。

C 的方向和 A、B 偏得比较明显，所以相似度就会低一些。

余弦相似度做的事，就是尽量排除“模长”的影响，只看两个向量的**方向夹角**。即：

- 夹角越小：方向越接近，语义越相似
- 夹角越大：方向差得越多，语义越不相似

一句话总结就是，向量里的数字决定了它在语义空间里的位置。余弦相似度主要比较两个向量的方向是否接近，而不是比较它们数值整体有多大。方向越接近，两段文本的语义越相似。

## 2、余弦相似度公式

余弦相似度的标准公式是：

```text
cosine_similarity(A, B) = (A · B) / (||A|| × ||B||)
```

其中：

- `A · B` 表示 A 和 B 的点积
- `||A||` 表示 A 的模长
- `||B||` 表示 B 的模长

如果把每一维都展开，就是：

```text
cosine_similarity(A, B)
= (a1×b1 + a2×b2 + ... + an×bn)
  ÷
  (√(a1² + a2² + ... + an²) × √(b1² + b2² + ... + bn²))
```

用人话说就是：

> 余弦相似度 = A 和 B 的点积 ÷（A 的模长 × B 的模长）

这张图可以先帮你建立一个整体感觉：

![[cosine-formula-breakdown.png]]

这里面只需要理解两个概念：**点积** 和 **模长**。

## 3、点积

点积就是：

> A、B 同位置的数字相乘，然后全部加起来。

比如：

```text
A = [0.5, 0.5]
B = [0.7, 0.7]
```

A 和 B 的点积是：

```text
A·B = a1×b1 + a2×b2 + ... + an×bn
```

```text
0.5 × 0.7 + 0.5 × 0.7
= 0.35 + 0.35
= 0.70
```

点积可以粗略理解为：两个向量在同一方向上有多少“重合”。不过要注意，点积不只受方向影响，也会受模长影响。所以余弦相似度后面还要再除以两个向量的模长。

## 4、模长

模长就是向量自己的长度。比如：

```text
A = [0.5, 0.5]
```

A 的模长就是：

```text
|A| = √(0.5² + 0.5²)
= √(0.25 + 0.25)
= √0.5
≈ 0.707
```

## 5、为什么要除以模长

因为余弦相似度想比较的是**方向**，不是单纯比较数字大小。

比如：

```text
A = [1, 1]
B = [10, 10]
```

B 的模长明显比 A 大很多，但它们方向一样。

所以公式里要除以两个向量的模长，把“模长差异”的影响抵消掉，最后重点看方向是否一致。


## 6、用几个向量实际看一下
假设现在有四个二维向量：

```text
A = [0.5, 0.5]
B = [0.7, 0.7]
C = [0.7, 0.5]
D = [-0.6, -0.5]
```

它们大概可以画成这样：

![[cosine-examples-ab-ac-ad.png]]

我们用 A 分别去和 B、C、D 做余弦相似度。

### A 和 B

```text
A = [0.5, 0.5]
B = [0.7, 0.7]
```

A 和 B 的方向完全一致，只是 B 更长一点。所以它们的余弦相似度是：1.000，这表示：**方向完全一致，非常相似。**

### A 和 C

```text
A = [0.5, 0.5]
C = [0.7, 0.5]
```

C 的方向和 A 有一点点偏差，但整体还是很接近。所以它们的余弦相似度大约是：0.986，这表示：**方向很接近，语义也很接近。**

### A 和 D

```text
A = [0.5, 0.5]
D = [-0.6, -0.5]
```

D 和 A 基本是反方向。所以它们的余弦相似度大约是：-0.996，这表示：**方向几乎相反，语义不相似。**

## 7、代码演算

上面的计算，用 Python 写出来其实很简单。

下面这段代码没有使用 `sum()`、`zip()` 这种压缩写法，而是把每一步都拆开写出来，方便你对照公式理解：

- `dot_product()`：计算点积
- `magnitude()`：计算模长
- `cosine_similarity()`：计算余弦相似度

```python
import math

def dot_product(vec1, vec2):
    """计算两个向量的点积"""
    if len(vec1) != len(vec2):
        raise ValueError("两个向量的维度必须相同")

    # 点积的计算方式：
    # 第 1 维相乘 + 第 2 维相乘 + ... + 第 n 维相乘。
    dot = 0

    for index in range(len(vec1)):
        # 取出两个向量在同一个位置上的数字。
        value_from_vec1 = vec1[index]
        value_from_vec2 = vec2[index]

        # 同位置的两个数字相乘。
        product = value_from_vec1 * value_from_vec2

        # 把每一维的乘积累加起来。
        dot = dot + product

    return dot


def magnitude(vec):
    """计算一个向量的模长"""
    # 模长的计算方式：
    # 每一维平方后相加，最后再开根号。
    sum_of_squares = 0

    for index in range(len(vec)):
        value = vec[index]

        # 当前维度的数字平方。
        square = value * value

        # 把每一维的平方累加起来。
        sum_of_squares = sum_of_squares + square

    # 对平方和开根号，得到向量的模长。
    return math.sqrt(sum_of_squares)


def cosine_similarity(vec1, vec2):
    """计算两个向量的余弦相似度"""
    # 第一步：计算两个向量的点积。
    dot = dot_product(vec1, vec2)

    # 第二步：分别计算两个向量的模长。
    vec1_magnitude = magnitude(vec1)
    vec2_magnitude = magnitude(vec2)

    if vec1_magnitude == 0 or vec2_magnitude == 0:
        raise ValueError("零向量没有明确方向，不能计算余弦相似度")

    # 第三步：按照公式计算余弦相似度。
    # 余弦相似度 = 点积 / (向量 1 的模长 * 向量 2 的模长)
    cosine_score = dot / (vec1_magnitude * vec2_magnitude)

    return cosine_score

# 用这几个二维向量测试计算
A = [0.5, 0.5]
B = [0.7, 0.7]
C = [0.7, 0.5]
D = [-0.6, -0.5]

for name, vec in [("B", B), ("C", C), ("D", D)]:
    # 用 A 分别和 B、C、D 比较方向是否接近。
    dot = dot_product(A, vec)
    a_magnitude = magnitude(A)
    other_magnitude = magnitude(vec)
    score = cosine_similarity(A, vec)

    print(f"A 和 {name}")
    print(f"  点积: {dot:.3f}")
    print(f"  A 的模长: {a_magnitude:.3f}")
    print(f"  {name} 的模长: {other_magnitude:.3f}")
    print(f"  余弦相似度: {score:.3f}")
    print()
```

代码里的 `:.3f` 只是控制输出格式，表示保留 3 位小数，不影响真实计算结果。

输出结果是：

```text
A 和 B
  点积: 0.700
  A 的模长: 0.707
  B 的模长: 0.990
  余弦相似度: 1.000

A 和 C
  点积: 0.600
  A 的模长: 0.707
  C 的模长: 0.860
  余弦相似度: 0.986

A 和 D
  点积: -0.550
  A 的模长: 0.707
  D 的模长: 0.781
  余弦相似度: -0.996
```

这段代码对应的就是公式：

```text
cosine_similarity(A, B) = (A · B) / (||A|| × ||B||)
```

所以：

- A 和 B 的方向完全一样，结果是 `1.000`
- A 和 C 的方向很接近，结果是 `0.986`
- A 和 D 的方向几乎相反，结果是 `-0.996`

代码里虽然没有画箭头，但它算的就是箭头之间的方向关系。

## 8、余弦相似度的结果怎么看

对于非零向量，余弦相似度的结果在 `-1` 到 `1` 之间：

```text
1    ：方向完全一样，非常相似
0    ：方向接近垂直，相关性不明显
-1   ：方向完全相反，非常不相似
```

放到 RAG 里，就是：用户问题会被转换成一个向量。知识库里的每个文本块，也会被同一个 embedding 模型转换成向量。

系统会计算：

> 用户问题向量 和 每个文本块向量 的余弦相似度。


一般来说，谁的分数越高，说明谁和用户问题的语义越接近。然后系统会把这些高分文本块取出来，交给大模型作为参考资料。
