# 一、通用基础大模型存在的问题
1. 基础大模型的数据是有时间节点的，模型训练好了以后对于最新产生的知识是没有自动更新能力的。
2. 基础大模型的训练是基于互联网上可以抓得到的数据的，对于私有领域的数据是没有办法为给基础大模型训练的，所以它不知道私有领域的知识。
3. 幻觉问题，私有领域的数据不足也是基础大模型产生幻觉的原因之一，也就是大家经常说的 大模型会一本正经的扯犊子。
4. 数据安全问题，这个不用多说了。

正式因为存在上面这些问题，所以RAG诞生了。
# 二、RAG是什么
RAG 全称是 **Retrieval-Augmented Generation**，中文一般叫 **检索增强生成**。说人话就是，让基础大模型回答问题前，先去你的资料库里查相关内容，再基于查到的内容回答。

比如你问：公司请假制度是什么？普通基础大模型可能凭训练记忆乱猜：干掉老板就随时可以请假。

而用了RAG后，它会先去检索你的公司文档，找到“请假制度”相关段落，再把这些段落交给大模型，让它总结回答。简单点说就是，RAG给大模型外挂一个可搜索的知识库，让它查资料后再回答：年假用光了，所以你不配请假。

##  without RAG：
基础模型没有外部资料，并且它仍然依赖自己之前训练时的数据，你问他这段空窗期发生的事，它不是不告诉你，而是真不知道。
## with RAG
这里要严肃的说一下
### 1. indexing： 建立索引
系统先把你扔给它的文档会切成很多小片，也叫chunks，然后把这些小片段转成向量embeddings。这样做是为了后面根据语义快速找到相关内容。
### 2. Retrieval：检索相关文档
用户输入的问题，比如图中问的是那个人为啥被裁员后又回来了（与降本增效无关）。
系统根据用户的问题，在已经建立好的向量索引里面搜索相关的文档片段，恰巧找到了几个与 Sam Altman 回归、董事会重组、人事动荡相关的 chunks。
### 3. Generation：生成回答
把用户输入的问题，和检索到的chunks，打包发给基础模型，因为基础模型有据可依，它重新组织一下好听的话再说给你听。

所以，有了RAG之后就能解决第一部分通用基础大模型存在的问题了。
