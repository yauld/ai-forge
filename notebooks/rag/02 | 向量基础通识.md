RAG工作流程中

1、知识和信息 =》 向量嵌入（向量化）=》 存入向量库
2、用户提问=》向量嵌入（向量化）=》 在向量库中匹配相关段落
3、匹配到了段落后，和prompt一起给大模型推理并返回。

这里面多次提到“向量”的概念，那么向量是什么

# 一、向量是什么

向量就是把一段文字翻译成的一串数字数组，用这串数字数组表示他的语义坐标位置。

比如问题：“产品A咋使用？”模型会把它变成一串数字数组，类似：[0.21, -0.08, 0.63, 0.15, ...]，这串数组就是向量。

文字是给人看的，而上面的向量是给机器看的，机器用它来确认 **“意思像不像”**

在 RAG 里，向量的作用是：**让机器能按语义找资料，而不是只按关键词找资料。**
比如知识库里有三段内容：

1. 产品 A 的安装步骤
2. 产品 A 的操作流程
3. 产品 B 的价格说明

用户问：产品A怎么使用？

哪怕第二段没有完全写“怎么使用”这四个字，只要它讲的是“操作流程”，它的向量就会和用户问题的向量比较接近。

所以系统就能判断：
**“产品 A 怎么使用” 和 “产品 A 的操作流程” 意思很近，应该把这段找出来。**

# 二、向量是咋来的

既然向量是数字数组，用户输入问题会被向量化成数字，知识信息也会被向量化成数字，那这个数字是咋来的？

流程很简单

1、把文字交个一个专门的模型，叫embedding model
2、这个模型去读懂文字的大概语义
3、计算并输出一串数字

```
“产品 A 怎么使用？”
        ↓
Embedding 模型
        ↓
[0.21, -0.08, 0.63, 0.15, ...]
```

**向量是 Embedding 模型把文字“翻译”成数字后的结果。** 

# 三、不同向量是咋匹配的

知识和信息会被切成多个文本块，每个文本块都有自己的向量；用户问题也会被转换成一个向量。

系统用用户问题的向量，去向量库里和这些文本块向量做相似度匹配，找出最相关的内容。
那么机器是怎么判断这个相似度的呢？如下图
![alt text](<assets/Pasted image 20260530091835.png>)

3段文字被embedding成3个向量，匹配的核心做法是先把文本变成向量后，在计算两个向量的相似度的分值，分数越接近1，语义越接近，而常用打分算法就是用余弦相似度。

在上面的例子中，A和B用余弦相似度打分结果是0.999... ，无限接近1了，即A和B几乎是一个意思。A和C的打分是0.3614... 无限不接近1，所以他们的不那么像。

也说明了另外一个问题，向量的匹配不是看字面是否一样，而是用数学分数判断两端文本在语义空间里里的近不近。

# 四、向量的语义角度

Embedding 模型会把一段文字变成一串数字，数字越多，能描述的语义角度就越丰富。
![alt text](<assets/Pasted image 20260530094031.png>)

```
[0.32, -0.81, 0.67, 0.14, ...]
```

这个向量里面：

- 0.32 是第 1 维的数值
- -0.81 是第 2 维的数值
- 0.67 是第 3 维的数值
- 0.14 是第 4 维的数值
- ...

如果一共有 **1536 个数字**，就是 **1536 维向量**。每一维可以粗略理解成一个“语义角度”的强弱分数。

生成维度越多，越能更好记录文本的语义特征，语义画像更细，做语义匹配时就越精准。

但是！

维度更多的代价就是，存储更大，计算和检索都会变重。实际选择的时候需要注意平衡。
